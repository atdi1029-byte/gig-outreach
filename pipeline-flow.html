How "Run Outreach" Works
User says "run outreach"
./discover.sh && ./discover.sh --taste && ./pipeline.sh --smart-picks
Phase 1: Discovery Chrome
Fetch past gigs from Google Sheet. Open each on Google Maps. Scrape "People also search for" recommendations. Filter, categorize, score, add to sheet. Find websites via Google search.
↓
Phase 2: Taste Discovery Taste
Broaden the search beyond the past gig network. 467+ search queries (9 venue types × 29 locations + keyword combos). Search Google Maps, pre-score results (40+ threshold), add matches to sheet. Picks up where it left off each run.
  • alex_taste.md — your taste in words (venue types + locations)
  • taste_venues.txt — 23 venues to learn from
  • taste_keywords.json — scraped keyword profile (built by --learn)
  • taste_queries.txt — completed query tracker
↓
Pipeline Mode? branch
--smart-picks → actionable untouched first, then ranked picks --untouched → only untouched venues --linkedin-retry → Step 4 only
↓
Phase 2: For Each Venue → run_venue()
Steps 1–4 execute sequentially per venue. Each step adds contacts to the Google Sheet. Every email is verified via ZeroBounce before saving. Sleep 30s between venues.
  • Step 1: Website scrape (Chrome JS inject)
  • Step 1B: Instagram Google search fallback
  • Step 2: Facebook + Instagram email scrape (Chrome)
  • Step 3: Apollo company search + people enrich (API)
  • Step 4: LinkedIn search + Apollo enrich (Chrome + API)
↓
Phase 3: Report + Status Update output
Generate HTML report (venues, contacts, flags, credits). Set venue status to "pipelined". Log everything to pipeline.log.
Taste Learn — discover.sh --learn
Read Venue Whitelist input
Load taste_venues.txt — 23 curated venues you love. Match against past gigs for location and score data.
↓
For Each Venue → Google Maps Chrome
Open venue on Google Maps. Click About tab. Scroll to load all content. Extract attributes via extract_venue_attributes.js.
  • 3 extraction strategies: section headers, icon-row attributes, aria-labels
  • Filters navigation items (nearby restaurants, hotels, etc.)
  • Returns: category, price level, attribute list
↓
Aggregate Keywords output
Count keyword frequency across all venues. Rank by how many venues share each trait. Save top 30 to taste_keywords.json.
  • Top keywords: cozy (9), romantic (7), upscale (6), great wine list (8)
  • Also tracks price levels and categories
Taste Search — discover.sh --taste
Generate Search Queries Python
Combine venue types × locations into prioritized queries. Skip already-completed queries (tracked in taste_queries.txt).
  • P1: Tier 1 types × Tier 1 locations (fine dining Georgetown DC)
  • P2: Tier 1 types × Tier 2 locations (winery Charlottesville VA)
  • P3: Tier 2 types × Tier 1 locations (boutique hotel Bethesda MD)
  • P4: Keyword-enhanced (cozy French restaurant McLean VA)
↓
For Each Query → Google Maps Chrome
Search Google Maps. Scroll results. Extract venue cards via extract_search_results.js. Capped at MAX_QUERIES per session.
↓
Pre-Score Each Result filter
Score 0-100 based on category tier, location, rating, reviews, state.
Score ≥ 40 → add to sheet Score < 40 → skip Skip: pubs, fast food, vacation rentals
↓
Add + Website Lookup Apps Script Chrome
Add venue to sheet with source="taste:query". Google search for website URL. Calculate driving distance.

Pipeline Entry — pipeline.sh
Select Mode entry point
--smart-picks (default) --untouched --linkedin-retry --batch / single venue
Smart Picks: Fetch + Rank ordering
First process untouched venues, sorted by actionable score (most ready first). Then fetch Smart Picks from Apps Script action=get_recommendations. Filter out already pipelined/contacted.
  • Phase 1 — Untouched: Sorted by actionable score (IG +3, website +2, contact form +2, state +1, city +1)
  • Venues already visible in app state folders go first
  • Capped by MAX_UT env var
  • Phase 2 — Smart Picks: Sort by recommendation_score DESC
  • Capped by MAX_SP env var
  • Sleep 30s between each venue
Step 1 — Website Scrape
Has website? gate
Yes → open in Chrome No website → skip to Step 1B
Chrome: Scrape Main Page Chrome JS
Inject JavaScript to extract emails, social links, and subpage URLs.
  • mailto: links — extract email + context (name/title)
  • Cloudflare protected emails — XOR decode data-cfemail
  • Visible text regex — emails in page body
  • All <a href> attributes — embedded emails
  • Schema.org JSON-LD — structured email field
  • Social links: Facebook + Instagram URLs
  • Subpage links: event, wedding, contact, about, music, etc.
↓
0 emails + has city? fallback
If no emails found on main page, try location-specific pages.
Try: /city-slug/, /locations/city-slug/ No city → continue
↓
Crawl Subpages (max 10) Chrome JS
Open each keyword-matching subpage, extract emails. Keywords: event, private, wedding, cater, contact, about, entertain, music, banquet, dining, party, book.
↓
Filter Junk Emails
Remove noreply@, support@, admin@, and junk domains (wix.com, wordpress, sentry.io, cloudflare, etc.). Skip emails > 60 chars.
↓
Verify via ZeroBounce API
Each email verified individually. 1s rate limit between calls.
valid / catch-all → save invalid / do_not_mail → discard unknown → save with status
↓
Push Contacts Apps Script
action=add_contact with name, title, email, source="website", verified status. Update venue social links (facebook, instagram, contact_form).
Step 1B — Instagram Fallback
Found Instagram in Step 1? gate
Already have IG → skip No IG → Google search fallback
Google: "Venue instagram" Chrome
Search Google for venue's Instagram profile. Extract first instagram.com URL via extract_ig.js. Save to venue record.
Step 2 — Social Media Scrape
Has Facebook or Instagram? gate
Has FB/IG → open in Chrome Neither → skip Step 2
Chrome: Scrape Social Pages Chrome JS
Open Facebook /about and /directory_contact_info pages. Open Instagram main page. Inject JS to extract emails from visible text. Filter junk domains.
social_scrape_emails.js
↓

Step 3 — Apollo API
Has Apollo API key? gate
Key set → proceed No key → skip Step 3
Company Search Apollo
Search by venue name. Fallback: search by website domain. Verify match (substring or word overlap ≥60%). Extract organization_id + primary_domain.
Match found → use Apollo domain No match → use website domain No domain at all → skip people search
↓
People Search Apollo
Search all people at company domain in region (MD, VA, DC, WV, PA, DE). Max 50 people (2 pages). Filter: only people with has_email=true.
↓
Bulk Enrich (batches of 10) Apollo
/people/bulk_match — reveals email + status. 1 credit per person. Skip unavailable emails.
  • Valid email → ZeroBounce verify → push (source="apollo")
  • No email → add as pending contact
  • Credits tracked in APOLLO_CREDITS_USED
Step 4 — LinkedIn + Apollo
Has API key + domain? gate
Both → proceed No key → skip No domain → add people as pending
LinkedIn Search (3 pages) Chrome
Search LinkedIn for venue name. Extract name, title, connection degree from each result.
  • Filter: current employees only (not "past:" or "former")
  • Filter: venue keyword must appear in title (≥1 word match)
  • Filter: skip already-known names
  • Retry up to 5x if page loads slowly
↓
Apollo Enrich by Name Apollo
For each LinkedIn person: first_name + last_name + domain → /people/bulk_match (batches of 10). 1 credit per person.
Email found → verify → push (source="linkedin+apollo") No email → add as pending No Apollo match → add from LinkedIn only

Taste System — How It Thinks
Layer 1: alex_taste.md
Your taste written in words. Describes dream venue types (French bistros, luxury hotels, historic fine dining...), sweet spot locations (Georgetown, Middleburg, Annapolis...), and what makes a gig great (educated audience, upscale ambiance, wine culture). This is the blueprint that defines the search space.
Written by hand — update anytime
↓
Layer 2: taste_venues.txt
23 venues you love, curated from your 70+ past gigs. This controls which venues --learn studies. Quality over quantity — only add venues you'd want more of.
Edit the list → re-run --learn
↓
Layer 3: --learn → taste_keywords.json
Visits each venue on Google Maps. Scrapes real attributes: cozy, romantic, upscale, great wine list, private dining, outdoor seating, fireplace, historic... Counts how many of your favorites share each trait. The data-driven refinement on top of the blueprint.
  • 30 keywords from 23 venues
  • Top: cozy (9), romantic (7), great wine list (8), upscale (6)
  • click_about_tab.js → extract_venue_attributes.js
↓
Layer 4: --taste → Search + Score + Add
Combines all three layers into 467+ Google Maps search queries. 9 venue types × 29 locations across MD, VA, DC, PA, DE, WV. Pre-scores every result (0-100) before adding to your sheet.
  • 9 venue types: fine dining, French, European bistro, winery, country club, private club, wine bar, luxury hotel, historic fine dining
  • 29 locations: Georgetown, Bethesda, Potomac, Great Falls, McLean, Middleburg, Annapolis, St. Michaels, Roland Park, Charlottesville, Wilmington...
  • Scoring: category tier (30pts), rating (20pts), location (10pts), sweet spot (10pts), state (10pts)
  • Threshold: score ≥ 40 to add, skips pubs/fast food/rentals
  • Runs MAX_QUERIES per session (default 20), resumes next time
